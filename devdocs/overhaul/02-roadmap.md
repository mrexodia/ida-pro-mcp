# ROADMAP
Roadmap synthesized from 18 audits into 7 dependency-ordered batches. The owner's headline asks - DEBUG-RE capabilities and the nested russian-doll call-hierarchy view - come early (Batches 3-4), but two foundational batches precede them because they directly unblock those features and lock the axis-7 never-patch-unless-asked invariant. Batch 1 establishes the consent/guardrail layer and fixes confirmed binary-mutation bugs. Batch 2 fixes call-edge/decompiler/error-contract foundations the russian-doll and trace features depend on. Batch 3 delivers real DEBUG-RE primitives (true return capture, x64 args, watchpoints, blocking control, stop-context, snapshot-diff). Batch 4 builds the nested call-hierarchy view (wide+fine, conditions+jumps) on the now-correct edges and ctree access. Batch 5 deepens static comprehension (ctree/microcode/lvar/types/decode correctness). Batch 6 does the architecture reorg + naming taxonomy + manifest registration once behavior is stable. Batch 7 closes discoverability/docs/tests. Cross-cutting de-duplicated themes: one shared bounded-traversal primitive, one decompiler cache, one large-output strategy, structured error taxonomy, and eval()->AST-whitelist hardening. Every binary-mutating capability is fenced behind explicit per-call consent + dry-run + original-byte capture before any new feature ships.

## Batch 1: Axis-7 consent layer + confirmed mutation-safety fixes
**Goal:** Lock the never-patch-unless-requested invariant into infrastructure (not per-tool discretion), fix confirmed binary-mutation bugs, and unify the safety taxonomy so metadata edits stay available while byte-writers are truly fenced. Lands first because every later feature touches mutating or debug-mutating surfaces and the only current gate is registry --unsafe.

- [axis 7 | val=high eff=medium risk=medium] Central server-side write/consent guard (require_consent + dry_run helper in core infra)
    Add a shared @mutates-style decorator / utils.guard_write(explicit) and a new PATCH/MUTATES_BINARY safety tier so patch/patch_asm/put_int refuse to write image bytes unless the call carried an explicit confirm token; default dry_run=True so the first call previews. http.py defaults every tool (incl. patchers) to enabled, so the gate must be server-side, not a UI hint.
- [axis 7 | val=high eff=medium risk=low] patch_asm/patch/put_int: capture+return original bytes, dry-run preview, full-span mapped check, length-mismatch handling
    Read get_bytes before patch_bytes and return original_bytes hex for reversibility; validate every byte of the span is mapped (not just start ea); warn/NOP-pad on a shorter assembled instruction leaving an orphan tail; add revert_patch + list_patches via get_original_byte/visit_patched_bytes.
- [axis 4 | val=high eff=low risk=low] Reclassify metadata edits WRITE vs byte-writers DESTRUCTIVE/PATCH consistently across code and docs
    rename/set_comments/append_comments/define_* are metadata-only but tagged DESTRUCTIVE, hiding them behind --unsafe with real patchers; reclassify to WRITE. Reserve DESTRUCTIVE/PATCH for image-byte writes. Fix the contradiction where tool-index is right but tools-reference/getting-started/decompiler-mastery/tool-authoring-guide label rename WRITE.
- [axis 7 | val=high eff=medium risk=medium] Gate binary-patching from injected Python (py_eval/py_exec_file)
    Add allow_patch=False default that wraps the namespace so ida_bytes.patch_bytes/idc.patch_* raise unless enabled, and auto-create an ida_undo point before any mutating script. py_exec_file also needs an allow-listed script root + size/encoding guards (currently any absolute host path is executable on a network-reachable server).
- [axis 7 | val=high eff=medium risk=low] Harden snapshot_restore + dbg_write: identity check, honest partial-failure, revert handle
    snapshot_restore writes live memory+regs, swallows all errors yet returns restored:True and never verifies same-process identity; record pid/module-base at save and refuse on mismatch, report partial. dbg_write should auto-snapshot original bytes and return a revert handle; fix its inconsistent missing-ok result shape.
- [axis 7 | val=medium eff=low risk=low] idb_save: bias is_idaq() exception to the SAFE GUI/no-kill branch and refuse silent overwrite
    Exception fallback currently biases to the headless DBFL_KILL branch (the #446 corruption it tries to avoid); flip to safe. Refuse save-as to an existing different path unless explicit overwrite=True.
- [axis 7 | val=medium eff=low risk=low] Make dbg_start entry-breakpoint planting opt-in and documented
    dbg_start silently auto-plants soft bps at every entry point when none exist - an undocumented IDB mutation; gate behind plant_entry_bps=False and document in RETURNS.
- [axis 8 | val=high eff=medium risk=low] Replace eval() probe-predicate sandbox claim with a real AST-whitelist evaluator
    _eval_predicate runs user strings via eval with only __builtins__ emptied (the noqa sandboxed comment is false; subclass-traversal escapes reach os/subprocess). Replace with ast.parse + node allowlist (Compare/BoolOp/BinOp, names limited to c/old/new + captured keys), used for predicates and conditions.
- [axis 7 | val=medium eff=low risk=low] Add a test encoding the axis-7 invariant + module-level READ-only assertions
    Test that every binary-mutating tool carries the byte-write tier + is registered unsafe, that a read-only workflow leaves IDB bytes unchanged, and that every @tool in api_analysis.py is READ-only. Lock guardrails into CI so a regression is visible.

## Batch 2: Foundational fidelity - call edges, decompiler cache, error contracts, traversal primitive
**Goal:** Fix correctness foundations the russian-doll view (Batch 4) and trace tools (Batch 3) depend on: consistent call-edge semantics, a single decompiler cache, structured error contracts, and one budgeted bounded-traversal primitive. Without these the headline axis-2 features inherit silent over/under-reporting and stale pseudocode.
**Depends on:** Batch 1: Axis-7 consent layer + confirmed mutation-safety fixes

- [axis 8 | val=high eff=medium risk=medium] Unify call-edge classification: chunk- and jmp-aware, call vs tailcall vs jump vs fallthrough vs indirect
    _direct_callees follows ALL CodeRefsFrom (counts tail-call jmps/jump-table targets as callees) while _direct_callers filters NN_call*, so the two directions are not transposes; this corrupts callees_recursive AND reaches. Add a shared classify_code_edge helper, walk func chunks (func_tail_iterator), resolve switch targets via get_switch_info, and surface unresolved indirect sites as explicit nodes.
- [axis 8 | val=high eff=medium risk=low] Single-pass decompile with a shared cfunc cache + invalidation
    decompile() decompiles twice (text then refs); analyze_batch and export_funcs add two more independent paths. Introduce a per-IDB cfunc cache keyed by func start_ea + dirty-counter in utils, invalidated on rename/retype. Wire force_recompile/mark_cfunc_dirty and auto-bump the dirty counter from modify/types edits so pseudocode is never stale.
- [axis 8 | val=high eff=low risk=low] Structured error taxonomy + consistent structured error fields
    Add NotFoundError/InvalidArgumentError/VersionUnsupportedError/FeatureUnavailableError subclassing McpToolError. Standardize tools that silently swallow errors (func_query bad regex, find_regex raw raise, entity_query bad min/max_addr, find string-mode swallow) onto a populated error field like search_text. Fix parse_address's misleading missing-0x message for not-found names.
- [axis 2 | val=high eff=medium risk=low] Budgeted recursive-walk primitive (deadline + node budget + cycle/SCC) shared by all graph tools
    Add walk_call_tree(ea, depth, node_budget, deadline=get_tool_deadline()) generator with cycle detection and back-edge flags so callgraph/callees_recursive/reaches and the new russian-doll tool share one deadline-aware bounded traversal. Standardize on BFS (callgraph's recursive DFS mislabels depth and risks RecursionError). Optional Tarjan SCC to collapse recursion clusters.
- [axis 8 | val=medium eff=low risk=low] Stabilize result schemas across success/error and add totals/truncation everywhere
    api_graph error branches drop max_depth/max_nodes/truncated; build results through one constructor. Add total + truncated to list_funcs/func_query/list_globals/imports/imports_query (only entity_query has total). Surface lookup_funcs '*' 1000-cap truncation. Disclose+parameterize all silent caps (5k-string, 500-site, 50-caller, 200/500 node/edge).
- [axis 8 | val=high eff=medium risk=medium] Fix immediate-search paging to linear next_start resumption
    find type=immediate rescans from each segment start every page with per-call seen_insn and cursor.next=offset+limit, giving quadratic cost and unstable offsets. Switch to insn_query-style next_start cursor resumption with persistent dedup within a scan.
- [axis 1 | val=medium eff=medium risk=low] Unify the two large-output strategies and harden output-limiting
    Collapse utils.handle_large_output (file-spill at 3000 lines) into rpc.py's truncation+download-URL path (50k chars) - one threshold, one envelope, one cache. Add a dict-key fan-out cap and switch the output cache from FIFO to LRU. Formalize the two registry tools/call monkeypatches into an ordered middleware chain (consent -> trace -> truncate -> handler).
- [axis 8 | val=high eff=low risk=low] Fix confirmed type-apply + parsing bugs (axis-8 sharp fixes)
    set_type passes PT_SIL (a parse flag) to apply_tinfo which expects TINFO_DEFINITE - applied types are non-definitive. type_query calls get_ordinal_limit() directly (breaks on IDA<8.4) while search_structs uses compat. int_convert auto-size raises spurious OverflowError for 0x80 and treats default size=64 as 64 bytes. declare_stack parses frame offset through parse_address (wrong for negative/decimal). Add post-apply re-read assertions.

## Batch 3: DEBUG-RE core - real captures, x64, watchpoints, blocking control, stop-context, snapshots
**Goal:** Deliver the owner's DEBUG-RE priority: turn the half-built dynamic toolkit into a faithful live-RE surface. Real return-value capture and correct x64 args are prerequisites for the russian-doll runtime overlay in Batch 4; blocking steps/run, memory watchpoints, stop-context, and snapshot-diff are the staple primitives flagged missing or fake. All @ext(dbg), read-only on the binary, debuggee mutations behind consent.
**Depends on:** Batch 1: Axis-7 consent layer + confirmed mutation-safety fixes

- [axis 2 | val=high eff=high risk=medium] Real return-value capture: true entry+return probe pairing
    trace_calls capture_ret and probe_net pre_post are FAKE - they only append an English instruction string; no return-site probe is installed and no value recorded. On the entry hit, auto-install a non-stopping one-shot return probe at [esp]/[rsp] (always returns False, self-disarms), joining entry+leave by (tid, rsp/call-id) and recording rax/eax. Load-bearing fix for In/Out trees.
- [axis 8 | val=high eff=medium risk=low] x64/auto calling-convention-aware argument capture
    _stack_arg reads args off the stack even when ptr_size==8, so Win64 (rcx/rdx/r8/r9) and SysV register args are garbage - every 64-bit probe mislabels arguments. Add a conv-aware resolver keyed off detected ABI / the function's IDB type, with shadow-space handling for arg4+.
- [axis 2 | val=high eff=medium risk=medium] Hardware breakpoints + data watchpoints (espions) with slot accounting
    dbg_add_bp hardcodes BPT_SOFT - no HW bps or data watchpoints at all. Add BPT_WRITE/BPT_RDWR/BPT_EXEC with size; surface type/size/hw-mode in dbg_bps. Add DR-slot accounting (report used/free, fail loudly on the 5th) and a software-watch fallback for ranges larger than one aligned slot, since watch_region silently misses writes outside its single slot.
- [axis 8 | val=high eff=medium risk=medium] Make resume/step blocking via the orphaned _continue_and_wait; fix async step correctness
    _continue_and_wait is dead code; step_into/step_over are async so returned post-step ip is racy/stale. Wire the helper into dbg_continue/dbg_run_to and add wait_for_next_event after step with an optional wait_ms. Fix run_until status corruption (any drained record forces status=hit, clobbering exited/timeout).
- [axis 2 | val=high eff=low risk=low] stop_context: one fused where-am-I tool
    After every stop an agent fires regs+stacktrace+read+disasm separately. Add stop_context returning ip/fn, disasm window, regs, decoded EFLAGS, stack args, stacktrace, and per-register pointer classification (which region/module each reg points into with a byte preview). Highest-leverage debug tool. READ.
- [axis 2 | val=high eff=low risk=low] memory_map + pointer classification
    _live_memory_ranges enumerates ranges internally but no tool surfaces them. Expose memory_map (regions with perms/module/type + module bases) and classify_pointer(addr). Foundational memory comprehension and feeds stop_context.
- [axis 2 | val=high eff=medium risk=low] snapshot_diff + auto-range + IDB-persistent snapshots
    Snapshots are in-process dicts with no diff, lost on restart. Add snapshot_diff (reg+range deltas via diff_buffers), auto_ranges mode (stack window + writable segments), and persist to the IDB netnode reusing trace.py plumbing. Snapshot-before/after a suspected mutator is the what-did-this-change primitive.
- [axis 2 | val=high eff=medium risk=medium] Register write, attach/detach, thread control, step-out, exception config
    Add dbg_set_reg (set_reg_val), dbg_attach/dbg_detach (dbg_exit currently always kills target), dbg_threads/dbg_select_thread, dbg_step_out (run-to-return), and exception_config (first-chance catch/pass) for packed/anti-debug targets. Replace hardcoded x86 GP register allowlist with a processor-aware set so ARM/AArch64/MIPS work.
- [axis 2 | val=medium eff=low risk=low] Counting/Nth-hit breakpoints, true every_nth sampling, per-probe record eviction
    Add break-on-Nth / count-only conditional breakpoints. Implement every_nth in _probe_dispatch (currently a no-op despite being advertised) via a per-probe skip counter. Evict a probe's ring records on probe_clear (currently orphaned). Remove or implement the inert buffer_mode param. Reconcile snapshot_delete/probe_arm DESTRUCTIVE mis-classification.
- [axis 8 | val=high eff=low risk=low] Add api_probes pure-helper unit tests + no-debugger-guard tests
    api_probes has ZERO coverage despite deterministic helpers (parse_capture_spec, parse_ptr_chain, build_probe_record, _stable_probe_id, validate_autopilot_step, plan_autopilot). Add unit tests for these and mock-based tests asserting _require_debugger() makes every probe/watch/trace entrypoint fail cleanly with no session. Add an opt-in headless live-debuggee smoke test (skip when no backend).

## Batch 4: Russian-doll nested call-hierarchy view (wide + fine, conditions + jumps)
**Goal:** The owner's flagship axis-2 capability: a zoomable nested In/Out call hierarchy giving both a wider (module/subsystem) and finer (intra-function branches/guards) view. Built on Batch 2's correct call edges + traversal primitive and able to overlay Batch 3's runtime data. Co-locates the scattered graph story (callees/callers/reaches/callgraph) into one owned module.
**Depends on:** Batch 2: Foundational fidelity - call edges, decompiler cache, error contracts, traversal primitive

- [axis 2 | val=high eff=medium risk=low] call_hierarchy: bidirectional depth-banded nested In/Out view rooted at a node
    New api_hierarchy.py tool: call_hierarchy(root, direction in/out/both, depth, max_nodes, include_indirect, exclude globs, collapse_leaves). Reuse the budgeted BFS helper for both directions with identical (now-unified) edge semantics; return signed depth-band levels arrays (the dolls), nodes with is_leaf/is_recursive/indirect_only, and edges tagged call/jump/tailcall/indirect. READ.
- [axis 2 | val=high eff=high risk=medium] function_skeleton: finer view - CFG with branch conditions and guarded calls
    Enrich basic_blocks: per-block terminating branch with human condition (jz->if(var==0)), true/false successors, calls_here, loop-back/back-edge markers, cyclomatic complexity. When decompiler available, pull ctree (cot_if/cot_for/cot_while) to attach real guard expressions and a guarded_calls list (decrypt fires under if(g_init && argc>1)). The conditions+jumps half of the russian-doll ask.
- [axis 2 | val=high eff=medium risk=low] Drill protocol: bounded outer view returns stable drill/expand cursors
    Every node/shell carries a drill handle {into:function_skeleton, addr} and expand {tool:call_hierarchy}; collapsed-leaf bundles and truncated shells carry continue cursors (mirroring basic_blocks next/done). Lets an agent zoom in (hierarchy node -> skeleton block -> guarded call) and out without dumping the whole closure.
- [axis 2 | val=medium eff=medium risk=low] module_hierarchy: subsystem/class-level russian-doll over a function set
    Rebuild analyze_component into a true nested view: auto-grow members from a seed (callees_recursive low depth, or class vtable/methods via domain_types), classify interface vs internal (fix the get_callers 50-cap/swallow mislabel), and emit inner intra-set graph + outer reaches-out (dependencies) + inner-reverse reached-by-in (API consumers) + shared globals. Drills into per-function call_hierarchy.
- [axis 2 | val=medium eff=high risk=medium] hierarchy_runtime_overlay: light up actually-taken edges/branches (ext dbg)
    Read-only overlay (sets bps/reads trace, never patches) merging probe/trace hit-counts onto a static call_hierarchy/function_skeleton by edge key, adding per-edge taken/hits and per-block executed. This guarded decrypt only fired when g_licensed!=0. Returns no_runtime_data when no session rather than implying coverage. Consumes Batch 3 real captures.
- [axis 2 | val=high eff=medium risk=low] Per-edge metadata + indirect-call surfacing + recursion/SCC representation
    GraphEdge gains call-site EA, count, indirect flag; unresolved indirect/virtual calls become explicit leaf nodes (vtable-heavy classes no longer silently sparse). Mark back-edges and optionally collapse mutually-recursive clusters into one SCC doll instead of exploding the fan.
- [axis 2 | val=high eff=medium risk=medium] Resolve indirect/virtual call edges via the decompiler/ctree where possible
    Add a decompiler-backed callee resolver walking cfunc cit_call nodes plus get_switch_info for jump-dispatch, unified with the mnemonic path, so the hierarchy/analyze tools recover indirect targets instead of dropping them. Prerequisite for faithful class/vtable hierarchies.
- [axis 5 | val=medium eff=low risk=low] eamap/boundaries bridge + russian-doll methodology doc
    Add map_ea_to_pseudocode / map_pseudocode_line_to_eas (hexrays eamap/boundaries) and attach pseudo_line ranges to skeleton blocks. Write docs/call-hierarchy-russian-doll.md wiring survey -> module_hierarchy -> call_hierarchy -> function_skeleton -> reaches -> runtime overlay, with depth/cap heuristics and the indirect-call caveat.

    Expose per-reference lvar access type (read/write/addr) and context (call_arg/comparison/condition) - replaces heuristic trace_data_flow with decompiler-accurate def-use. Add set_lvar writing name/type/comment via modify_user_lvar_info/save_local_variable_info (currently only rename exists) so the read-then-annotate loop is complete. WRITE (metadata).
- [axis 2 | val=high eff=medium risk=low] Structured decompile output: line<->address map, lvars, warnings
    Beyond inline /*0xNNNN*/ markers, return {line_no:[eas], ea:line_no}, the lvar list (name/type/location), and cfunc warnings so the model knows when pseudocode is unreliable and can pivot programmatically. Refs visitor currently only handles cot_obj - broaden.
- [axis 2 | val=medium eff=medium risk=low] Microcode access (text + resolved call args)

- [axis 8 | val=high eff=high risk=low] Type-aware value decoding (read_struct/get_string/get_global_value/read_struct_live)
    read_struct decodes ALL members as unsigned ints; branch on is_floating/is_signed/is_bool/is_enum, recurse one level into nested UDTs, decode arrays element-wise, extract bitfields by bit offset/width. get_string/global char-arrays hardcode UTF-8/type 0 - use get_str_type and decode per STRTYPE_*. Fix get_global_value inverted char-array condition. Honor DB endianness in get_int/put_int and read_struct_live overlay.
- [axis 2 | val=high eff=medium risk=low] Enum member inspection + bitfield-aware layout + typedef target resolution
    type_inspect/type_query report enum member_count 0; enumerate edm_t members (name/value/mask).     Reimplement off the legacy idc enum shim onto BTF_ENUM/edm_t; pass correct bitmasks instead of bmask=-1 (which mis-creates bitfield members and breaks value-conflict detection). Add mode (skip|update|error) for member value updates. Centralize any remaining shim in compat.

    infer_types mislabels heuristic guess_tinfo as method=hexrays/confidence=high; relabel and add string (is_strlit)/pointer/code-ref/array-of-struct detection so a known ASCII string isnt reported uint8_t[N]. Add preview/dry-run to set_type/type_apply_batch (resolve+report current->new without applying). Make set_type kind inference explicit/cheaper (dont decompile on every addr+name edit).
- [axis 2 | val=medium eff=high risk=medium] Field-level struct editing, array stamping, and til import/list
    Add add/rename/retype individual struct members and stamp a type across N elements (array) instead of forcing whole-decl redeclaration. Expose add_til / list loaded type libraries so WinAPI/libc structs can be loaded - a core 9.x typing workflow absent today. declare_type should return the names of types actually created.

    Document verb_noun + capability-prefix rules (bare verbs for static default namespace; dbg_*; one shared family for probe/watch/snapshot/trace/appcall; py_*/recipe_*; list_<noun> vs <noun>_query vs <noun>_inspect for the three read patterns).
